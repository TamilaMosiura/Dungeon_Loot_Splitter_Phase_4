Sync lifecycle

  • Very straightforward process where I fetch what is in state (if any) and update UI. if no data, than do nothing, as there is nothing to render
  • When data exists, show banner and render details
  • When user makes changes, validate changes and save to state. Then next time when page refreshed, the data coma back from server and can be rendered again.
  • To clean the state, there is reset button exists. It not clearly remove the state (as I don't know how to do that), rather set it to empy array for loot and set 0 to partySize




Load lifecycle

  • Loading was described above in sync. Anyway, I want to add here, that by loading errors might happen.
  • Errors can be - in data - this is when the data was saved incorrectly or the data itself is incorrect
  • it can be in connection - when some connection issues happening, or internet disappered
  • it can be error on server - data is not available and can be fetched correctly.
  • So, all of the situations should be handled correctly and in most cases shouwn to user with good UI
  • In some cases data can not be shown to user and system can retry a few times before really show to user that error happened
